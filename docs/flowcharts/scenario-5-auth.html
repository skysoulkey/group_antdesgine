场景5：用户登录与权限控制

用户从登录认证到角色权限分配的完整流程，包括 MFA 验证、强制改密、动态菜单过滤与数据隔离。

1

登录认证

用户在登录页输入手机号或邮箱，获取短信/邮件验证码后提交登录。系统校验凭证是否匹配。

首次登录强制改密

若账号标记 must_change_password: true，MFA 通过后系统强制跳转到修改密码页面，用户必须设置新密码后才能进入系统。

登录完成后，系统根据用户所属角色自动跳转到对应的默认首页。不同角色看到不同的入口页面。

菜单过滤规则：每个菜单项绑定所需权限模块（如"公司管理"需 company:manage）。用户登录后，前端获取角色对应的权限列表，逐项匹配。若子菜单全部被过滤，则父菜单自动隐藏。
数据隔离规则：集团级角色可查看所有公司数据；公司级角色仅可查看自己所属公司的数据。后端 API 通过 company_id 字段自动过滤。
