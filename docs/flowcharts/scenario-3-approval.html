自动审批规则匹配（分叉点）

新的待审批消息进入后，系统按以下优先级匹配审批规则：

优先级 1：企业专属规则 — 优先匹配该企业的独立规则（每企业一条）
优先级 2：全局兜底规则 — 未配置专属规则时，匹配全局规则（按币种 PEA / USDT 分别配置）

匹配到规则后，按以下条件判断：
① 匹配上规则 + 单笔金额在上限内 + 余额充足 + 累计未超总投入上限 → 自动通过
② 匹配上规则 + 余额不足 → 转为手动审批
③ 匹配上规则 + 超过总投入金额上限 → 转为手动审批，并提醒「超过总投入上限」
④ 未匹配上规则（未配置 / 未启用）→ 转为手动审批

启用校验：开启规则时校验配置项是否完整，若单笔上限或总投入上限为空则不予开启，提示「请先完善配置」。

5

三渠道通知分发 — 同时触达审批人

系统同时通过机器人、邮件、站内三个渠道发送审批通知。

通知对象（谁能看到消息）= 所有有审批权限的用户 + 群管理员手动拉入的用户
审批权限（谁能点按钮操作）= 只有审批权限列表中启用的审批人

投资审批群创建时机：当通知配置中已配置机器人通知地址，且首次出现待审批消息时，系统自动创建投资审批群。后续审批消息复用已创建的群。
群聊无法解散，机器人不开放公开搜索，群聊仅通过系统自动创建。
不考虑群主踢掉有审批权限的人的场景（被踢后该用户无法在群内审批）。
