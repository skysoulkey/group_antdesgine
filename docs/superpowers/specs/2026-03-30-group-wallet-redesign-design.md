# 集团钱包改造 Design Spec

**日期：** 2026-03-30
**作者：** Miya + Claude

---

## 背景与目标

现有集团钱包采用自助转账模式（自由填写收款账号 + 转账通讯录），不符合合规要求。改造后：

- 入金/出金必须通过预绑定的固定账号进行，账号信息由对接外部平台同步，本系统只读展示
- 出金引入后台审批流程，状态由外部系统回传
- 移除转账通讯录、二维码充值、月度图表、税费计算等旧功能
- 操作入口增加 MFA 二次验证保障安全

---

## 权限范围

- 仅 `group_admin`（集团管理员）可使用入金/出金功能
- 页面本身权限不变（`/finance/wallet` → `group_admin`）

---

## 页面结构

```
集团钱包（/finance/wallet）
├── 余额区
│   ├── USDT 余额卡片
│   └── PEA 余额卡片
├── 绑定账号卡片
│   ├── 账号信息（只读，从外部平台同步）
│   ├── [修改绑定账号] 跳转链接
│   └── 操作按钮：[入金]  [出金]
└── 订单记录
    ├── 筛选栏：类型 / 状态 / 日期范围
    └── 订单表格
```

**移除内容：**
- 转账通讯录（左侧地址簿）
- 二维码充值弹窗
- 月度转出统计图表
- 税费计算逻辑（5% 税率、免税额度）

---

## 订单记录表格

### 列定义（按顺序）

| 列名 | 说明 |
|------|------|
| 发起时间 | 订单创建时间 |
| 结束时间 | 订单完结时间（待审批时为空） |
| 订单号 | 系统生成唯一编号 |
| 类型 | 入金 / 出金 |
| 币种 | USDT / PEA |
| 金额 | 数字，右对齐 |
| 状态 | 待审批 / 成功 / 失败（见下方状态说明） |
| 备注 | 用户填写的备注 |

### 筛选项

- **类型**：全部 / 入金 / 出金
- **状态**：全部 / 待审批 / 成功 / 失败
- **日期范围**：RangePicker，按发起时间筛选

---

## 订单状态定义

统一枚举值：`待审批` / `成功` / `失败`

### 入金流程
```
用户发起入金 → 生成订单（状态：成功 或 失败）
```
入金无需审批，提交即终态。

### 出金流程
```
用户发起出金 → 生成订单（状态：待审批）
                     ↓ 外部系统审批
              成功 / 失败（状态回传更新）
```

### 状态 Tag 颜色
- `待审批` → warning（橙）
- `成功` → success（绿）
- `失败` → error（红）

## 入金弹窗

### 触发
点击绑定账号卡片上的 [入金] 按钮，弹窗从按钮位置展开（`mousePosition` 动效），关闭时缩回按钮位置。

### 第一步：填写信息
```
绑定账号：[xxxx-xxxx-xxxx]（只读展示）  [修改绑定账号 →]
币种：    [USDT ▼] / [PEA ▼]
金额：    [          ]
备注：    [          ]（选填）

                      [下一步]
```

点击「修改绑定账号」：关闭当前弹窗，跳转至修改绑定账号页面。

### 第二步：确认 + MFA 验证
```
── 确认信息 ──────────────────────
绑定账号：xxxx-xxxx-xxxx
类型：    入金
币种：    USDT
金额：    1,000.00
备注：    xxxxxx
──────────────────────────────────

MFA 验证码：[      ]

            [返回修改]  [确认提交]
```

MFA 验证码正确后方可提交。提交后生成订单，初始状态为「成功」或「失败」。

---

## 出金弹窗

结构与入金弹窗完全一致，差异仅在提交后订单初始状态为「待审批」。

---

## 修改绑定账号页面

独立页面（路由待定，建议 `/finance/wallet/bind-account` 或在基础信息页增加入口）。

### 页面内容
```
当前绑定账号：[xxxx-xxxx-xxxx]（只读）

新账号信息：
账号：     [          ]
备注名称： [          ]（选填）

MFA 验证码：[      ]

            [取消]  [保存]
```

保存成功后返回集团钱包页，绑定账号展示刷新为新值。

---

## 绑定账号卡片

账号信息字段（从外部平台同步，只读）：
- 账号 ID / 账号名称（具体字段以对接协议为准，mock 阶段用占位数据）
- 同步时间戳（可选展示）

---

## Mock 数据说明

前端 mock 阶段：
- 绑定账号：硬编码占位值，模拟外部同步结果
- 订单记录：生成 10-15 条覆盖全状态的 mock 数据（入金成功/失败、出金待审批/成功/失败）
- MFA 验证：任意 6 位数字视为通过（mock 不做真实校验）
- 出金订单状态变更：通过表格行内操作模拟外部回传（如添加「模拟审批」按钮，仅开发调试用）

---

## 不在本次范围内

- 外部平台账号同步接口对接
- 真实 MFA 验证逻辑
- 真实审批回调接口
- 修改绑定账号的权限管控（由后端接口控制）
