## 1. 概述

### 1.1 背景

企业在经营过程中会触发各类事务（追加投资、释放股份、下拨、调回等），这些事务需要通知到公司侧相关人员。其中涉及资金的事务（追加投资、释放股份）需要公司主或经营管理员进行审批后才能生效。

### 1.2 设计目标

- 建立事件驱动 + 状态机的审批流程，职责清晰、状态可追溯
- 支持小程序、邮件、站内三种渠道的通知分发，审批类通知在小程序渠道带 Inline Keyboard 操作按钮
- 支持公司级自动审批规则配置，命中规则的事务跳过人工审批
- 在 Web 端提供审批管理页面（`/finance/approvals`），包含审批列表和审批规则两个 Tab

### 1.3 核心架构

采用**事件驱动 + 状态机**方案：

```
企业事务发生（10 种事件类型，完整列表见《消息通知需求文档》第六节）
  │
  ├─ 纯通知类（8 种）→ 直接发通知（三渠道）
  │
  └─ 需审批类（2 种：追加投资 / 释放股份）
       → 生成审批单（带类型、金额、截止时间）
       → 匹配自动审批规则
         → 命中：状态直接置为「自动通过」→ 发通知（纯消息，三渠道）
         → 未命中：状态置为「待审批」→ 发通知（小程序带 Inline Keyboard 操作按钮）
           → 人工审批（小程序按钮 / Web 端操作）→ 状态变更 → 发结果通知（三渠道）
           → 超时未处理 → 状态置为「超时拒绝」→ 发结果通知（三渠道）
```

## 2. 审批事件模型

### 2.1 需审批的事件类型

本文档仅关注需要审批的 2 种事件类型。其余 8 种纯通知类事件的定义和模版见《消息通知需求文档》。

| 事件类型 | 说明 | 小程序按钮 |
|----------|------|-----------|
| 持股企业追加投资 | 企业要追加投资，公司需出资 | ✅ 同意 / ❌ 拒绝 |
| 持股企业释放股份 | 企业释放股份，公司需购买 | ✅ 同意 / ❌ 拒绝 |

### 2.2 审批状态机

5 种状态，区分「谁做的决定」和「怎么做的决定」：

```
                 ┌─ 命中自动审批规则 ──→ 【自动通过】──→ 发通知（纯消息）
                 │
事件到达 → 规则匹配 ┤
                 │              ┌──→ 【已通过】（人工）──→ 发结果通知
                 └─ 未命中 ──→ 【待审批】──┼──→ 【已拒绝】（人工）──→ 发结果通知
                               发审批通知   └──→ 【超时拒绝】──→ 发结果通知
```

状态值：`pending` / `approved` / `rejected` / `auto_approved` / `timeout_rejected`

## 3. 审批单数据结构

```
审批单 {
  id:                   唯一标识
  eventType:            事件类型（追加投资 / 释放股份）
  status:               待审批 / 已通过 / 已拒绝 / 自动通过 / 超时拒绝

  // 触发方信息
  sourceCompanyId:      触发方企业 ID
  sourceCompanyName:    触发方企业名称
  sourceOwnerId:        企业主 ID
  sourceOwnerUsername:  企业主用户名
  sourceOwnerNickname:  企业主昵称

  // 业务数据 —— 追加投资
  totalInvestAmount:    追加投资总金额
  shareRatio:           本公司占股比例
  investCurrency:       需要投资货币单位
  investAmount:         需要投资货币金额

  // 业务数据 —— 释放股份
  releaseRatio:         释放股份比例
  totalShareValue:      总股本预估金额
  releaseCurrency:      释放股份货币单位
  releaseAmount:        释放股份金额

  // 时效
  createdAt:            审批开始时间
  deadline:             审批截止时间（由触发方定义，超时自动拒绝）

  // 审批信息
  approvedBy:           审批人昵称（人工审批时记录）
  approvedById:         审批人 ID（人工审批时记录）
  approvedAt:           审批时间
  remark:               审批备注（可选）

  // 归属
  companyId:            所属公司 ID
  companyName:          所属公司名称
}
```

业务数据字段按 `eventType` 区分，追加投资填充投资相关字段，释放股份填充股份相关字段。

## 4. 通知与审批的协作关系

> 通知模版文案、渠道配置、通知对象管理的完整定义见《消息通知需求文档》。本节仅描述审批流程与通知系统的交互方式。

### 4.1 审批相关的通知触发场景

| 触发时机 | 小程序渠道 | 邮件渠道 | 站内渠道 |
|----------|-----------|----------|----------|
| 需审批事务到达且未命中自动规则 | Inline Keyboard 按钮（同意/拒绝） | 含跳转 Web 审批页面链接 | 带跳转审批页面入口 |
| 需审批事务到达且命中自动规则 | 普通消息 | HTML 邮件 | 通知列表 |
| 审批结果产生（已通过/已拒绝/超时拒绝） | 普通消息 | HTML 邮件 | 通知列表 |

通知对象的确定：直接读取通知管理（`/system/notifications`）中的通知配置，审批模块不重复维护通知对象。

### 4.2 审批操作入口

审批类事务未命中自动规则时，通过以下入口完成人工审批：

| 入口 | 操作方式 | 说明 |
|------|---------|------|
| 小程序 Inline Keyboard | 点击「✅ 同意」或「❌ 拒绝」按钮 | 直接在小程序内完成审批，按钮点击后置灰不可再操作 |
| Web 端审批列表 | 点击「通过」或「拒绝」，二次确认弹窗 | `/finance/approvals` 页面操作，可填写备注 |
| 邮件跳转 | 点击邮件内链接跳转 Web 审批页面 | 邮件内不直接操作 |
| 站内信跳转 | 点击站内信入口跳转 Web 审批页面 | 站内信不直接操作 |

### 4.3 模版变量

## 5. 自动审批规则配置

### 5.1 规则归属

每个**公司**独立配置自己的自动审批规则，公司之间互不影响。

### 5.2 规则数据结构

```
自动审批规则 {
  id:          唯一标识
  companyId:   所属公司 ID
  name:        规则名称
  eventType:   适用事件类型（追加投资 / 释放股份）
  enabled:     是否启用

  // 条件组（AND 关系：所有条件同时满足才命中）
  conditions: [
    {
      field:     匹配字段（amount / sourceCompanyId / ...）
      operator:  比较方式（≤ / ≥ / = / 包含于 / ...）
      value:     阈值或目标值
    }
  ]
}
```

### 5.3 支持的条件类型

| 条件字段 | 适用事件 | 操作符 | 示例 |
|----------|----------|--------|------|
| 金额（investAmount / releaseAmount） | 两种都适用 | ≤、≥ | 金额 ≤ 10000 自动通过 |
| 触发方企业（sourceCompanyId） | 两种都适用 | 指定企业列表 | 企业 A、B 的申请自动通过 |
| 金额 + 企业组合 | 两种都适用 | 同时满足 | 企业 A 且金额 ≤ 5000 自动通过 |

### 5.4 规则匹配逻辑

```
事件到达 → 查找该公司下所有「启用中」的规则
        → 按事件类型过滤
        → 逐条匹配条件（单条规则内条件之间 AND）
        → 任一规则命中 → 自动通过
        → 全部未命中 → 进入人工审批
```

多条规则之间 **OR 关系**（任一命中即可），单条规则内多个条件 **AND 关系**（全部满足才命中）。

### 5.5 审批规则页面（`/finance/approvals` 审批规则 Tab）

**规则列表**：

表格列：

| 列 | 说明 |
|----|------|
| 创建时间 | 规则创建时间 |
| 更新时间 | 规则最后更新时间 |
| 规则名称 | |
| 适用事件类型 | 追加投资 / 释放股份 |
| 适用范围 | 适用的企业范围 |
| 触发条件 | 金额 / 企业 / 金额+企业 |
| 启用状态 | |
| 操作 | 编辑、启用/停用、删除 |

**新增/编辑规则弹窗**：
- 输入规则名称
- 选择事件类型（追加投资 / 释放股份）
- 动态添加条件行：选字段 → 选操作符 → 填值
- 多行条件之间为 AND 关系
- 启用开关

## 6. Web 端审批管理页面

### 6.1 页面位置

路由：`/finance/approvals`，归属公司管理模块，侧边栏名称「投资审批」。

导航位置：侧边栏「公司管理」下，与公司持股、公司收益并列。审批人（公司主/经营管理员）本身就有公司管理的权限，权限天然对齐。

### 6.2 Tab 结构

| Tab | 说明 |
|-----|------|
| 审批列表 | 默认 Tab，展示所有审批单（待处理 + 历史） |
| 审批规则 | 自动审批规则配置（见第 5 节） |

Tab 切换使用 `useSearchParams` 读写 `?tab=` 参数，面包屑联动。

### 6.3 审批列表 Tab

**筛选区域**：

| 筛选项 | 控件类型 | 说明 |
|--------|----------|------|
| 事件类型 | Radio.Button 平铺 | 追加投资 / 释放股份（≤6 枚举值用平铺） |
| 审批状态 | Radio.Button 平铺 | 全部 / 待审批 / 已通过 / 已拒绝 / 自动通过 / 超时拒绝 |
| 触发方企业 | Select 下拉 | 企业数量可能 >6，用下拉 |
| 时间范围 | DateRangePicker | |

**表格列**：

| 列 | 说明 |
|----|------|
| 审批开始 | 审批单创建时间 |
| 审批截止 | 截止时间 |
| 事件类型 | 追加投资 / 释放股份 |
| 企业名称 | 触发方企业名称 |
| 企业 ID | 触发方企业 ID |
| 企业主昵称 | 触发方企业主昵称 |
| 企业主 ID | 触发方企业主 ID |
| 企业主用户名 | 触发方企业主用户名 |
| 归属公司 | 审批所属公司名称 |
| 归属公司 ID | 审批所属公司 ID |
| 业务摘要 | 根据事件类型展示关键金额信息 |
| 审批状态 | 待审批 / 已通过 / 已拒绝 / 自动通过 / 超时拒绝 |
| 审批人昵称 | 人工审批时显示 |
| 审批人 ID | 人工审批时显示 |
| 审批时间 | |
| 操作 | 待审批：通过/拒绝；其他：查看详情 |

**操作交互**：
- 点击「通过」：二次确认弹窗，可填写备注（选填），确认后状态变更
- 点击「拒绝」：二次确认弹窗，可填写拒绝理由（选填），确认后状态变更
- 点击「查看详情」：弹窗展示审批单完整信息

**详情弹窗内容**：
- 触发方信息：企业名称、企业 ID、企业主昵称、企业主 ID、企业主用户名
- 业务数据：根据事件类型展示对应字段（投资金额/股份比例等）
- 审批信息：审批状态、审批人昵称、审批人 ID、审批时间、备注
- 时效信息：审批开始时间、审批截止时间
- 归属信息：归属公司、归属公司 ID

## 7. 小程序审批交互（Inline Keyboard）

> 按钮样式和模版文案见《消息通知需求文档》第五节。

### 7.1 审批操作流程

```
用户收到小程序通知（带 Inline Keyboard）
  ├─ 点击「✅ 同意」
  │    ├─ 按钮文字变为「已同意 ✓」，按钮置灰不可再点
  │    ├─ 调用审批 API → 后端更新审批单状态为「已通过」
  │    └─ 系统自动发送「审批操作结果 — 通过」通知（三渠道同步）
  │
  ├─ 点击「❌ 拒绝」
  │    ├─ 按钮文字变为「已拒绝 ✗」，按钮置灰不可再点
  │    ├─ 调用审批 API → 后端更新审批单状态为「已拒绝」
  │    └─ 系统自动发送「审批操作结果 — 拒绝」通知（三渠道同步）
  │
  └─ 截止时间内未操作
       ├─ 按钮自动移除，消息追加「⏰ 已超时」
       ├─ 后端更新审批单状态为「超时拒绝」
       └─ 系统自动发送「审批操作结果 — 超时」通知（三渠道同步）
```

### 7.2 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 审批单已被其他人处理 | 回复：「该申请已由 XXX 处理」，按钮置灰 |
| 审批单已超时 | 回复：「该申请已超时自动拒绝」，按钮移除 |
| 无审批权限的人点击 | 按钮不展示，或回复：「您无权限操作」 |
| 余额不足导致审批通过后执行失败 | 审批状态仍为「已通过」，另发「余额不足」通知 |

## 8. 审批人权限

### 8.1 有权审批的角色

- 公司主（company_owner）
- 公司经营管理员（company_operator）

### 8.2 审批决策模式

任一有权人审批即生效。不存在多级审批或会签机制。多个有权人同时收到审批通知，谁先处理算谁的。

### 8.3 超时机制

- 审批截止时间由触发方（企业）在生成事务时定义
- 超时未处理 → 自动拒绝（`timeout_rejected`）
- 超时后触发结果通知，通知公司侧相关人员
