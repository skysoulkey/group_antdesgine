# 系统日志需求文档

> 页面路径：设置中心 → 系统日志（`/system/logs`）

---

## 一、页面结构

系统日志页面采用 **Tabs 切换**，分为两个标签页：

| Tab | 说明 |
|-----|------|
| 登录日志 | 记录用户登录、登出、登录失败事件 |
| 操作日志 | 记录用户在系统内的关键业务操作和页面访问行为 |

### 权限与数据隔离

| 角色层级 | 可见范围 |
|---------|---------|
| 集团用户（group_owner / group_audit） | 本集团全部日志（含下属公司） |
| 公司用户（company_owner / company_audit） | 仅本公司日志 |

---

## 二、登录日志

### 2.1 登录流程与记录节点

用户从打开登录页到进入系统，完整流程如下，每个节点均需记录日志：

```
输入账号密码 → 验证账号密码 → MFA 验证 → 首次改密判断 → 进入系统
                    │             │              │
                密码错误        验证失败       改密完成/失败
                账号不存在

登录成功 + MFA通过 → 记录「登录」
用户主动退出 → 记录「登出」
密码错误/账号不存在/MFA失败 → 记录「登录失败」
首次改密提交 → 记录「首次改密」
```

### 2.2 记录场景（共 4 种）

| 序号 | 操作类型 | 触发时机 | 结果 | 角色字段 |
|------|---------|---------|------|---------|
| 1 | 登录 | 账号密码正确 + MFA 验证通过 | 成功 | 显示用户实际角色 |
| 2 | 登出 | 用户主动点击退出按钮 | 成功 | 显示用户实际角色 |
| 3 | 登录失败 | 账号不存在 / 密码错误 / MFA 验证失败 | 失败 | 显示 `-`（无法确定角色） |
| 4 | 首次改密 | 首次登录触发强制改密流程，用户提交新密码 | 成功 / 失败 | 显示用户实际角色 |

**补充说明：**
- 登录成功后，若触发首次改密，此时用户尚未进入系统，登录日志已记录，改密结果另记一条
- 登录失败需区分失败原因：账号不存在、密码错误、MFA 验证码错误、MFA 超时
- 首次改密成功后自动进入系统，改密失败停留在改密页面

### 2.3 记录字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 时间 | 事件发生时间 | 2025-11-20 08:30:00 |
| 账号 | 登录账号（登录失败时显示 `-`） | Miya / `-` |
| 角色 | 用户角色（登录失败时显示 `-`） | 集团主 / `-` |
| 操作 | 登录 / 登出 / 登录失败 / 首次改密 | 登录 |
| 登录IP | 客户端 IP 地址 | 104.28.200.1 |
| IP归属地 | IP 所属国家/地区 | 🇸🇬 新加坡 |
| 结果 | 成功 / 失败 | 成功 |
| 失败原因 | 仅失败时记录 | 账号不存在 / 密码错误 / MFA验证码错误 / MFA超时 / 改密不符合规则 |

### 3.4 所属模块枚举

| 模块 | 覆盖范围 |
|------|---------|
| 仪表盘 | 集团仪表盘、公司仪表盘 |
| 用户管理 | 创建/编辑/删除/查看用户、重置密码 |
| 公司管理 | 创建/编辑公司、公司收益、订单 |
| 企业管理 | 企业详情查看、邀请码、解散企业 |
| 集团金融 | 资金下拨/调回、钱包充值/转出、绑定账号、集团收益 |
| 通知管理 | 通知对象增删改、通知渠道开关 |
| 系统设置 | 安全设置、通用设置、通知配置、个人中心、系统日志 |
