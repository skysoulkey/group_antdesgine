# 系统日志需求文档

> 页面路径：设置中心 → 系统日志（`/system/logs`）
> 面向研发，描述登录日志与操作日志的完整记录场景和字段规范。

---

## 一、页面结构

系统日志页面采用 **Tabs 切换**，分为两个标签页：

| Tab | 说明 |
|-----|------|
| 登录日志 | 记录用户登录、登出、登录失败事件 |
| 操作日志 | 记录用户在系统内的关键业务操作，附带操作对象详情 |

### 权限与数据隔离

| 角色层级 | 可见范围 |
|---------|---------|
| 集团用户（group_owner / group_audit） | 本集团全部日志（含下属公司） |
| 公司用户（company_owner / company_audit） | 仅本公司日志 |

---

## 二、登录日志

### 2.1 记录场景

| 序号 | 操作类型 | 触发时机 | 结果 | 角色字段 |
|------|---------|---------|------|---------|
| L1 | 登录 | 用户输入正确账号密码 + 通过 MFA 验证后 | 成功 | 显示用户实际角色 |
| L2 | 登出 | 用户主动点击退出按钮 | 成功 | 显示用户实际角色 |
| L3 | 登录失败 | 账号不存在 / 密码错误 / MFA 验证失败 | 失败 | 显示 `-`（无法确定角色） |

### 2.2 字段定义

| 字段 | 说明 | 示例 |
|------|------|------|
| id | 日志唯一标识 | `LL0000001` |
| loginTime | 事件发生时间 | `2025-11-20 08:30:00` |
| username | 登录账号（失败时为输入的账号） | `Miya` / `unknown_user` |
| roles | 用户角色（登录失败时显示 `-`） | `集团主` / `-` |
| action | 操作类型：登录 / 登出 / 登录失败 | `登录` |
| loginIp | 客户端 IP 地址 | `104.28.200.1` |
| country | IP 归属地 | `新加坡` |
| result | 结果：成功 / 失败 | `成功` |

### 2.3 筛选条件

- 操作类型（登录 / 登出 / 登录失败）
- 角色
- 关键词搜索（账号名 / IP）

## 三、操作日志

### 3.1 记录场景

#### 用户与权限

| 序号 | 操作行为 | 所属模块 | 触发时机 | detail 示例 |
|------|---------|---------|---------|------------|
| O1 | 创建用户 | 用户管理 | 提交创建用户表单 | `{ targetUser, assignedRoles, level, company }` |
| O2 | 编辑用户角色 | 用户管理 | 修改用户角色并保存 | `{ targetUser, oldRoles, newRoles }` |
| O3 | 删除用户 | 用户管理 | 确认删除用户 | `{ targetUser, roles }` |
| O4 | 禁用/启用用户 | 用户管理 | 切换用户启用状态 | `{ targetUser, action: "禁用"/"启用" }` |
| O5 | 重置密码 | 用户管理 | 管理员为用户重置密码 | `{ targetUser }` |

#### 公司管理

| 序号 | 操作行为 | 所属模块 | 触发时机 | detail 示例 |
|------|---------|---------|---------|------------|
| O6 | 创建公司 | 公司管理 | 提交创建公司表单 | `{ companyName, region, currency }` |
| O7 | 编辑公司信息 | 公司管理 | 修改公司资料并保存 | `{ companyId, changedFields }` |
| O8 | 变更公司状态 | 公司管理 | 启用/停用公司 | `{ companyId, companyName, oldStatus, newStatus }` |

#### 资金操作

| 序号 | 操作行为 | 所属模块 | 触发时机 | detail 示例 |
|------|---------|---------|---------|------------|
| O9 | 集团下拨 | 集团金融 | 集团向公司划转资金 | `{ from: "集团钱包", to, amount, currency }` |
| O10 | 集团调回 | 集团金融 | 从公司调回资金至集团 | `{ from, to: "集团钱包", amount, currency }` |
| O11 | 绑定钱包账号 | 集团金融 | 新绑定或修改钱包绑定账号 | `{ walletType, action: "绑定"/"修改" }` |

#### 企业管理

| 序号 | 操作行为 | 所属模块 | 触发时机 | detail 示例 |
|------|---------|---------|---------|------------|
| O12 | 邀请企业 | 企业管理 | 提交企业邀请 | `{ enterpriseName, inviteCode }` |
| O13 | 企业认证审核 | 企业管理 | 审核通过/驳回企业认证 | `{ enterpriseId, enterpriseName, action: "通过"/"驳回" }` |
| O14 | 企业解散 | 企业管理 | 确认解散企业 | `{ enterpriseId, enterpriseName }` |

#### 通知管理

| 序号 | 操作行为 | 所属模块 | 触发时机 | detail 示例 |
|------|---------|---------|---------|------------|
| O15 | 新增通知配置 | 通知管理 | 创建新的通知规则 | `{ notificationType, channels, enabled }` |
| O16 | 编辑通知配置 | 通知管理 | 修改通知规则并保存 | `{ notificationType, changedFields }` |
| O17 | 删除通知配置 | 通知管理 | 确认删除通知规则 | `{ notificationType }` |

#### 个人操作

| 序号 | 操作行为 | 所属模块 | 触发时机 | detail 示例 |
|------|---------|---------|---------|------------|
| O18 | 修改密码 | 个人中心 | 用户修改自己的密码 | `{ username }` |

#### 数据导出

| 序号 | 操作行为 | 所属模块 | 触发时机 | detail 示例 |
|------|---------|---------|---------|------------|
| O19 | 导出数据 | （按实际页面） | 点击导出按钮 | `{ exportType, dateRange, format }` |

### 3.2 字段定义

| 字段 | 说明 | 示例 |
|------|------|------|
| id | 日志唯一标识 | `OL0000001` |
| operateTime | 操作时间 | `2025-11-20 09:15:00` |
| username | 操作人账号 | `Miya` |
| roles | 操作人角色 | `集团主` |
| operation | 操作行为（参见 3.1） | `创建用户` |
| module | 所属模块 | `用户管理` |
| operateIp | 客户端 IP 地址 | `104.28.200.1` |
| result | 结果：成功 / 失败 | `成功` |
| detail | 操作对象详情（JSON） | `{ targetUser: "Tom", assignedRoles: ["company_ops"] }` |

### 3.3 筛选条件

- 所属模块
- 结果（全部 / 成功 / 失败）
- 关键词搜索（账号名 / 操作行为 / IP）

### 3.4 详情查看

操作日志表格每行提供「详情」按钮，点击弹窗展示 `detail` 字段的 JSON 内容，供管理员查看操作对象的完整信息。

## 五、前端现状说明

当前前端已实现：
- Tabs 切换（登录日志 / 操作日志）
- 登录日志支持登录、登出、登录失败三种类型，失败时角色显示 `-`
- 操作日志支持 detail 字段，表格提供「详情」弹窗展示 JSON
- 数据隔离（集团看全部，公司看本公司）
- 筛选与搜索

以上数据均为前端 Mock，后端接入时需提供对应的 API 接口，日志写入由后端在各业务接口中完成。
